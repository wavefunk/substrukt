- [ ] **`src/uploads/mod.rs`**: Add `pub fn generate_derivative(source_path: &Path, params: &TransformParams) -> eyre::Result<Vec<u8>>`:
  - Load image with `image::open`
  - Apply resize/crop based on `FitMode`:
    - `Contain`: `img.resize(w, h, FilterType::Lanczos3)`
    - `Cover`: resize to cover, then crop centered on focal point
    - `Scale`: `img.resize_exact(w, h, FilterType::Lanczos3)`
  - No upscaling: if requested dimensions exceed original, use original dimensions
  - Cap at 4096x4096
  - Encode in original format (detect from source MIME)
  - Return bytes

### Commit 9: Transform serving route

- [ ] **`src/routes/uploads.rs`**: Modify `serve_upload` and `serve_upload_no_name` handlers to accept `Query<TransformQuery>` (new struct with `w`, `h`, `fit`, `focal` fields). The current `serve_file` helper (`routes/uploads.rs:391-394`) doesn't have access to query params -- it needs to be updated to accept transform params, or the logic moves into the handler itself. Also update `serve_upload_by_hash` (`routes/uploads.rs:339-389`) to accept optional transform params:
  - Parse query params: `w`, `h`, `fit` (default "contain"), `focal` (default center)
  - If no params: serve original (existing behavior, no change)
  - If params present and MIME is raster image (`image/jpeg`, `image/png`, `image/gif`, `image/webp`):
    1. Reject if dimensions > 4096 (400)
    2. Reject if MIME is `image/svg+xml` or non-image (400)
    3. Resolve focal point: if `focal=auto`, load from DB; if explicit, parse; else center
    4. Compute derivative hash
    5. Check derivative cache: if exists, serve from disk
    6. If not: generate derivative, write to `_derived/`, serve
    7. ETag = `"{params_hash}"`
- [ ] For animated GIF: `image` crate decodes first frame only -- derivative is static. Document in error response header or serve with warning

- [ ] **`src/sync/mod.rs`**: Update export bundle (line 50-55) to exclude `_derived` from uploads:

  - **Recommended**: explicit iteration approach, as it keeps bundles lean for large media sites
