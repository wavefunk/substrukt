## Commits

### Commit 1: Shared query pipeline in content module

- [ ] **`src/content/mod.rs`**: Add types:
  ```rust
  #[derive(Debug, Clone, Default)]
  pub enum SortOrder { #[default] Asc, Desc }

  pub struct QueryParams {
      pub status: String,       // "published", "draft", "all"
      pub q: String,            // text search
      pub filters: Vec<(String, String)>,  // field-specific filters
      pub sort_field: String,   // field name, default "_id"
      pub sort_order: SortOrder,
      pub offset: usize,
      pub limit: Option<usize>, // None = all
  }

  pub struct QueryResult {
      pub entries: Vec<ContentEntry>,
      pub total: usize,         // total after filtering, before pagination
  }
  ```
- [ ] **`src/content/mod.rs`**: Add `pub fn query_entries(entries: Vec<ContentEntry>, params: &QueryParams) -> QueryResult`:
  1. `filter_by_status(entries, &params.status)`
  2. `filter_by_fields(entries, &params.filters)` (new)
  3. `filter_entries(entries, &params.q)` if q is non-empty
  4. Count `total = entries.len()`
  5. Sort by `params.sort_field` with `params.sort_order`, secondary sort by entry ID for stability
  6. Apply offset/limit: `entries.into_iter().skip(params.offset).take(limit).collect()`
  7. Return `QueryResult { entries, total }`
- [ ] **`src/content/mod.rs`**: Add `pub fn filter_by_fields(entries: Vec<ContentEntry>, filters: &[(String, String)]) -> Vec<ContentEntry>`:
  - For each entry: check all filters (AND logic)
  - For each filter `(field, value)`: get `entry.data.get(field)`, compare:
    - String: exact match (case-sensitive)
    - Number/Integer: parse filter value as f64/i64, compare
    - Boolean: parse filter value as bool, compare
    - Missing field: no match
  - Return entries that match all filters
- [ ] **`src/content/mod.rs`** (sort helper): Add `fn sort_entries(entries: &mut [ContentEntry], field: &str, order: &SortOrder)`:
  - Extract field value from each entry's data
  - Compare: strings lexicographically, numbers numerically, booleans (false < true)
  - Missing values sort last
  - Apply secondary sort by `entry.id` for stability
- [ ] **Unit tests**: `query_entries` with various param combos: search + filter, sort + paginate, empty results, offset beyond total

### Commit 3: API pagination with envelope response

- [ ] **`src/routes/api.rs`**: Extend `ListParams` (lines 69-77):

- [ ] **`src/routes/api.rs`**: Add helper to extract filters from `extra`:
  ```rust
  fn extract_filters(extra: &HashMap<String, String>) -> Vec<(String, String)> {
      extra.iter()
          .filter_map(|(k, v)| k.strip_prefix("filter.").map(|f| (f.to_string(), v.clone())))
          .collect()
  }
  ```
- [ ] **`src/routes/api.rs`**: Add constant `const MAX_API_LIMIT: usize = 500;`
- [ ] **`src/routes/api.rs`**: Update `list_entries` handler (line 288+):
  1. Build `QueryParams` from `ListParams`: parse sort/order, extract filters, clamp limit to MAX_API_LIMIT
  2. Call `content::query_entries(entries, &params)`
  3. Apply transforms (resolve references, render markdown, strip status) on paginated results only
  4. If `limit` or `offset` was specified: return envelope `{ "data": [...], "meta": { "total", "limit", "offset", "count" } }`
  5. If neither specified: return bare array (backwards compatible)
- [ ] **`src/routes/api.rs`**: Update ETag handling: envelope responses should not be cached in `etag_cache` (compute per-response, same as current list behavior)

- [ ] **Critical**: Check that `serde(flatten)` with `extra: HashMap` works with Axum's `Query` extractor, which uses `serde_urlencoded`. There are known compatibility issues: `serde_urlencoded` may not fully support `flatten` with HashMap. If it doesn't work, fall back to manually parsing `filter.` params from the raw query string via `axum::extract::RawQuery` or switch to a custom extractor. Test this early (before writing all the filter logic) to avoid a late redesign.
