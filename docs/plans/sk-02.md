## Commits

### Commit 1: `save_entries_batch` in content module

- [ ] **`src/content/mod.rs`**: Add `save_entries_batch(content_dir, schema, entries: Vec<(Option<&str>, Value)>) -> eyre::Result<Vec<Result<String, eyre::Error>>>`
  - For `StorageMode::Directory`: iterate entries, call `save_entry` individually, collect results
  - For `StorageMode::SingleFile` with `Kind::Collection`: load file once, apply all inserts/updates in memory (same logic as `save_entry` single-file branch but batched), write once, return all IDs
  - For `StorageMode::SingleFile` with `Kind::Single`: reject (single-kind cannot batch)
- [ ] **`src/content/mod.rs`**: Add `generate_entry_id_with_collision_check(schema, data, existing_ids: &HashSet<String>) -> String` -- wraps existing `generate_entry_id`, checks result against `existing_ids`, appends `-{uuid_first_8}` on collision
- [ ] **`src/content/mod.rs`** (tests): Unit test `save_entries_batch` for directory mode (3 entries, verify all files created)
- [ ] **`src/content/mod.rs`** (tests): Unit test `save_entries_batch` for single-file mode (3 entries, verify single file contains all)
- [ ] **`src/content/mod.rs`** (tests): Unit test collision detection (two entries with same title, second gets UUID suffix)

### Commit 2: Bulk API request/response types

- [ ] **`src/routes/api.rs`**: Add types at module level:
  ```rust
  const MAX_BULK_ENTRIES: usize = 500;

  #[derive(serde::Deserialize)]
  struct BulkCreateRequest { entries: Vec<serde_json::Value> }

  #[derive(serde::Deserialize)]
  struct BulkUpdateRequest { entries: Vec<serde_json::Value> }

  #[derive(serde::Deserialize)]
  struct BulkIdsRequest { ids: Vec<String> }

  #[derive(serde::Serialize)]
  struct BulkItemResult {
      #[serde(skip_serializing_if = "Option::is_none")]
      index: Option<usize>,
      #[serde(skip_serializing_if = "Option::is_none")]
      id: Option<String>,
      status: String,
      #[serde(skip_serializing_if = "Option::is_none")]
      errors: Option<Vec<String>>,
      #[serde(skip_serializing_if = "Option::is_none")]
      error: Option<String>,
  }

  #[derive(serde::Serialize)]
  struct BulkResponse {
      total: usize,
      #[serde(flatten)]
      counts: std::collections::HashMap<String, usize>,
      results: Vec<BulkItemResult>,
  }
  ```
- [ ] Add helper `fn bulk_error_response(status: StatusCode, msg: &str) -> impl IntoResponse` for common 400/404 patterns

### Commit 3: API bulk create endpoint

- [ ] **`src/routes/api.rs`**: Add `async fn api_bulk_create(...)` handler:
  1. `require_api_role(&token, "editor")`
  2. `require_token_app(&token, &app)`
  3. Load schema, reject `Kind::Single` with 400
  4. Validate `entries` array: non-empty, <= `MAX_BULK_ENTRIES`
  5. Collect existing entry IDs from cache (prefix `{app_slug}/{schema_slug}/`)
  6. For each entry: validate via `validate_content`, generate ID with collision check, on success call `save_entry`, reload cache, track upload references via `uploads::extract_upload_hashes` + `db_update_references` (upload fields must reference pre-existing hashes from `POST /uploads` -- binary data cannot be embedded in JSON bulk payloads)
  7. For `StorageMode::SingleFile`: use `save_entries_batch` instead of per-entry `save_entry`
  8. Build `BulkResponse` with per-item results
  9. Audit log: `content_bulk_create` with detail showing counts
- [ ] **`src/routes/api.rs`**: Register route in `api_app_routes()` -- add `.route("/content/{schema_slug}/_bulk/create", post(api_bulk_create))`. `_bulk` is a literal segment so Axum routes it before the `{entry_id}` wildcard regardless of registration order. Group with other bulk routes for readability.

### Commit 4: API bulk update endpoint

- [ ] **`src/routes/api.rs`**: Add `async fn api_bulk_update(...)` handler:
  1. Role/app check
  2. Load schema, validate request
  3. For each entry: extract `_id`, verify entry exists, snapshot current via `history::snapshot_entry`, validate, save
  4. For `StorageMode::SingleFile`: batch snapshots first, then `save_entries_batch`
  5. Reload cache per entry, track upload references
  6. Audit log: `content_bulk_update`
- [ ] Register route: `.route("/content/{schema_slug}/_bulk/update", post(api_bulk_update))`

### Commit 5: API bulk delete endpoint

- [ ] **`src/routes/api.rs`**: Add `async fn api_bulk_delete(...)` handler:
  1. Role/app check
  2. Load schema
  3. For each ID: `db_delete_references`, `delete_entry`, `delete_history`, remove from cache
  4. Entries not found: mark as error in results
  5. Audit log: `content_bulk_delete`
- [ ] Register route: `.route("/content/{schema_slug}/_bulk/delete", post(api_bulk_delete))`

### Commit 6: API bulk publish/unpublish endpoints
